云端部署（长期在线）

架构建议

  • 1 台 Linux VPS 常驻运行 gateway
  • 密钥使用环境变量或受控配置
  • 日志、备份、健康检查独立保留

基础流程

# 1) 安装 OpenClaw # 2) 配置 openclaw.json # 3) 启动并设为守护进程 openclaw gateway start openclaw status --deep

守护进程与开机自启

按系统平台文档使用 LaunchAgent/systemd。目标是服务异常后自动拉起，重启后自动恢复。

公网暴露策略

  • 优先只绑定内网或 loopback，再通过反代暴露
  • 开启鉴权，限制来源 IP
  • 如果走反代，配置 trusted proxies

openclaw security audit --deep openclaw status --deep openclaw update status
