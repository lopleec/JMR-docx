OpenClaw 专区（macOS / Linux）

这页是完整部署教程：从安装 OpenClaw，到改 JMR API，再到重启验证。

1）安装 OpenClaw

# 方式一：一键脚本（推荐）
curl -fsSL https://openclaw.ai/install.sh | bash

# 方式二：npm 全局安装
# 安装 brew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装 Node.js
brew install node@24

# 安装 OpenClaw
npm install -g openclaw@latest

2）运行配置向导

openclaw onboard

配置向导会引导你完成：

  • 选择 AI 模型提供商（Anthropic / OpenAI / Gemini / Ollama 等）
  • 配置 API Key
  • 选择消息渠道（Telegram / WhatsApp / iMessage / 飞书等）
  • 安装守护进程（Gateway 作为系统服务运行）
JMR API 建议：向导阶段先选 OpenAI 并填一个占位 key，后面我们用配置文件覆盖。

3）编辑配置文件

文件路径：

~/.openclaw/openclaw.json

把下面 3 个输入填好，系统会自动生成可直接复制粘贴的完整配置：

3.5）替换范围（重点）

不是全替换。
你只需要在 ~/.openclaw/openclaw.json 里，找到原有配置中类似 { “models”:{ … } 的那一段， 把它替换成上面生成的配置块（从 "models" 到 "agents.defaults.workspace"）。 其他无关配置保持不动。 
# 仅替换这一段（示意）
"models": { ... },
"agents": {
  "defaults": {
    "model": { ... },
    "workspace": "..."
  }
}

# 不要整文件覆盖。
# 其他 channel / tools / memory 等原配置保持不变。

4）重启 Gateway

openclaw gateway restart

5）验证是否完成

openclaw status --deep
openclaw models list --json
完成 ✅
安装 → 向导初始化 → 替换 JMR 配置 → 重启 → 验证。
