OpenClaw 部署方式（简版）

适合个人 / 家庭服务器 / 小团队，快速上线

1）本机部署（最快）

  • 场景：自己电脑上直接跑，调试最方便。
  • 步骤：安装 OpenClaw → 配置通道（如 iMessage / Telegram）→ 启动 gateway。
  • 优点：上手快；缺点：电脑关机就离线。

2）云服务器部署（稳定）

  • 场景：希望 24/7 在线、可远程访问。
  • 步骤：准备 Linux VPS → 安装 OpenClaw → 配置密钥与通道 → 设守护进程。
  • 优点：稳定在线；缺点：有服务器成本。

3）家庭局域网部署（隐私优先）

  • 场景：家里小主机/NAS 常开，数据尽量本地化。
  • 步骤：内网机器安装 OpenClaw → 绑定消息渠道 → 按需做外网转发。
  • 优点：私有化强；缺点：外网访问配置稍复杂。
推荐：先本机验证功能，再迁移到云端长期运行。
备注：生产环境建议加监控、备份和最小权限策略。
