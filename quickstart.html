快速开始（10 分钟）

目标：先让 OpenClaw 跑起来并可交互，再考虑优化和加固。

1. 安装与检查

openclaw --version openclaw status

如果 `status` 正常返回，你的基础环境就 OK。

2. 启动 Gateway

openclaw gateway start openclaw gateway status

确认服务是 running 状态。

3. 配置模型与通道

至少要保证一个可用模型和一个可用消息通道（例如 iMessage / Telegram）。

openclaw status --deep

这里能看到模型可用性、通道健康状态、错误提示。

4. 跑一次端到端测试

  • 发一条文字消息，看是否能正常回复
  • 如果要识图，发一张图片做冒烟测试
  • 如果你走 iMessage，建议同时测一份文件附件

5. 快速定位问题

  • 模型不可用：先看 key 与模型能力声明
  • 图片不可识别：确认 `imageModel` 和本地媒体目录限制
  • 通道异常：先看 `openclaw status --deep` 的 channel health
